## Customer model

### Persona 1 — Maya, engineering manager

**Today (without this CLI):** Maya keeps three Linear tabs open at all times — the current cycle board, the active project page, and the triage inbox — plus a Friday-update Google Doc, Slack, and a scratch tab where she pastes screenshots of cycle progress bars. For her Friday update she eyeballs the cycle board to estimate completion %, scrolls per-assignee for overload signals, and manually re-types "what slipped" from comparing yesterday's cycle view against today's. There is no view in Linear that answers "what's the cycle-over-cycle trend over the last 8 cycles" without exporting CSVs and pivoting them in Sheets.

**Weekly ritual:** Friday stakeholder update (cycle progress %, shipped, slipped, projected landings, blocked work) and Monday sprint planning where last-cycle velocity should inform rebalance.

**Frustration:** Linear gives her every number she needs but never in the *shape* the update needs them. Every Friday is N copy-pastes from N tabs into one doc.

### Persona 2 — Devon, staff engineer mid-project

**Today (without this CLI):** Day starts at `linear.app/issues?assignee=me&state=!Done`, then he flips to the project page for blocked work, then to the triage tab for incoming bugs, then to a terminal where he hand-types `git checkout -b devon/ENG-1234-thing` after copying the issue ID out of Linear. He creates pp-test issues mid-debug and forgets to clean them up — they accumulate as cruft on the cycle board until the EM @s him.

**Weekly ritual:** Every weekday: scan personal queue, pick up unblocked work, push PRs, link them to Linear, file or claim incoming bugs.

**Frustration:** Linear shows him who is blocking him on each issue page, but never the inverse — "what work is stuck because *I* haven't merged it" — as a personal queue. He has to click into each of his assigned issues to discover his downstream impact.

### Persona 3 — Priya, product manager doing portfolio review

**Today (without this CLI):** Owns three initiatives spanning seven projects across four teams. For Tuesday portfolio review she opens the initiative page (a rollup of static project target-dates set whenever someone created the project), then opens each of seven project pages to eyeball the actual progress, then opens the milestone view to see which dates have slipped. Linear's initiative health is whatever the project creator typed in months ago, not a projection from current velocity.

**Weekly ritual:** Tuesday portfolio review with leadership; Wednesday: triage feature requests against initiative fit.

**Frustration:** "Which milestone in my portfolio is most at risk this month?" requires opening seven project pages and doing the math in her head, every Tuesday.

### Persona 4 — Sam, agent operator driving Linear via MCP

**Today (without this CLI):** Daily Claude Code prompt: "what should I work on today?" The current third-party Linear MCP exposes 60+ endpoint-mirror tools — agent burns ~3K tokens enumerating before it can call anything, then has to chain `listIssues(assignee=me)` + `listCycles(active=true)` + `listIssueRelations(...)` + locally re-rank. Friday cleanup prompt — "delete the test issues you made this week" — has no contract: the agent does not know which tickets it created vs which existed before.

**Weekly ritual:** Daily "what now," Friday "what did I make and undo any leftovers," ad-hoc triage and sprint plan via MCP.

**Frustration:** The agent loses the plot mid-ritual because the surface is too thin for joined workflows and too token-heavy for a small workspace.
